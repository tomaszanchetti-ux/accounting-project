## Decisiones tomadas en esta sesión

### 1. Stack del MVP

Se confirmó el stack costo-eficiente del proyecto:

- `Next.js`
- `FastAPI`
- `Supabase Postgres`
- `Supabase Storage`
- `Vercel`
- explicación `rule-based + template-based`

### 2. Lógica de ejecución

Se fijó que:

- la unidad real de trabajo es la **Card**
- la ejecución debe ser `task por task`
- se trabaja siempre primero en local
- luego se valida
- luego se hace commit
- luego push

### 3. Git workflow

Se fijó que:

- se trabaja en `branch por epic`
- `main` no se toca directamente
- el merge a `main` ocurre solo con QA final del branch completo

### 4. Disciplina de continuidad

Se fijó que:

- `AGENTS.md` es lectura obligatoria al empezar
- cada sesión debe crear un `ARD_WS[N]_DDMMAAAA.md`
- los documentos madre deben actualizarse cuando cambian las reglas de trabajo
