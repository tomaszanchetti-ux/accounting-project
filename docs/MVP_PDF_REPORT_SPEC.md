# MVP PDF Report Spec

## Objetivo

Definir el alcance del PDF del MVP simplificado para que el exportable sea
útil, sobrio y rápido de implementar.

## Decisión de alcance

El primer PDF del MVP será:

- `por concepto`
- exportado desde el `Deep Dive`
- orientado a revisión ejecutiva y sharing rápido

No entra todavía:

- PDF multi-concepto
- PDF por run completa
- branding por cliente
- tablas largas multipágina con todo el dataset

## Estructura del reporte

El PDF debe incluir:

1. título del reporte
2. contexto de la run
3. contexto del concepto
4. KPIs principales del concepto
5. summary statement
6. recommended action
7. top causes
8. evidencia resumida
9. muestra breve de registros

## Tono visual

El PDF debe sentirse:

- sobrio
- claro
- ejecutivo
- fácil de escanear

Evitar:

- decoración excesiva
- densidad visual innecesaria
- narrativa larga

## Nombre del archivo

Convención inicial:

- `reconciliation-report-<run-label>-<period>-<concept-code>.pdf`

## Criterio de éxito

El PDF del MVP está bien si:

- se descarga desde el `Deep Dive`
- refleja fielmente la pantalla
- puede compartirse como artefacto de revisión
- mantiene legibilidad en una lectura rápida
